Mobile User Interface Design Process - From Paper Prototyping to Interactive Prototyping

This is my first attempt of designing through the process of prototyping

I split the process into three parts:

  1. Paper Prototyping
  2. Static Design Prototyping
  3. Interactive Prototyping

Paper Prototyping


Paper prototyping is a good process to employ at the early stage of the UI design. The designs are easily disposable and composable.
Through this process, I learned how my UI will generally look like, what components I would have in my design, and what each components would do.
Most importantly understanding how the users can navigate between the different screens of the user interface.


Static Prototyping

After defining components, actions, and navigations through paper design, the next process is to mock up the design digitally.
A powerful tool to create digital UI design is using Adobe XD

**The fonts used in the design above, Google Product Sans, are liscensed by Google. The font used in the design is solely for education purpose.

Interactive Prototyping

We have digitally mockup the static screens from the process above. However, the result is static. We can add interactions and interactivity in AdobeXD to stimilate what the UI will be like and how the users would navigate arond the screen before implementing them through codes.
